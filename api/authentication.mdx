---
title: "Authentication"
description: "API key authentication for the OPS Public API."
---

All API requests must include a team-scoped API key as a Bearer token in the `Authorization` header.

```bash
curl https://app.oneperfectslice.ai/api/public/v1/slices \
  -H "Authorization: Bearer sk_your_api_key"
```

## API keys

API keys are prefixed with `sk_` and scoped to a single team. A key for Team A cannot access Team B's data.

### Creating a key

<Steps>
  <Step title="Open Org Settings">
    Navigate to [app.oneperfectslice.ai/settings](https://app.oneperfectslice.ai/settings) and open the **API Keys** section. You'll need Org Admin permissions.
  </Step>
  <Step title="Create a key">
    Click **Create API Key**. Provide a name (e.g., "Production" or "MCP Server") and select the team it should be scoped to.
  </Step>
  <Step title="Copy the key">
    Your key will be displayed **once**. Copy it immediately and store it securely (e.g., a password manager or environment variable).
  </Step>
</Steps>

<Warning>
  The raw key value is never stored by OPS — only an Argon2id hash. If you lose it, you'll need to create a new one.
</Warning>

### Managing keys

From the API Keys page you can see all keys for your organization:

- **Name** — the label you assigned
- **Team** — which team the key is scoped to
- **Last used** — when the key was last used for an API call
- **Revoke** — permanently disable the key

## Authentication errors

| Code | HTTP | Meaning |
|------|------|---------|
| `MISSING_TOKEN` | 401 | `Authorization` header with Bearer token is required |
| `INVALID_TOKEN` | 401 | Token is not recognized |
| `TOKEN_REVOKED` | 401 | Token has been revoked in Org Settings |
| `TOKEN_EXPIRED` | 401 | Token has passed its expiration date |

## Security details

- Keys are stored using **Argon2id** hashing
- Token verification uses constant-time comparison to prevent timing attacks
- Keys are team-scoped — cross-team access is not possible
- Resources belonging to a different org return `404`, not `403` (no existence leakage)
- Revoked keys are immediately invalidated

## Best practices

- Create separate keys for different use cases (e.g., one for your integration, one for MCP)
- Rotate keys periodically
- Never commit keys to source control or expose them in client-side code
- Use environment variables in your applications
- Revoke keys immediately if they may have been compromised
